# Task B — Development Log

This file documents how I used Lovable for Task B.

## Step 1 — Starting Idea

I started with the CampusClubHub idea from Task A.

The goal was to make an app for a student club.

I chose this because the assignment suggested an app for a club as a possible middle-sized pet project.

## Step 2 — Using Lovable

I opened Lovable and used a prompt to describe the app.

I asked Lovable to create a student club management app with events, members, announcements and sign-ups.

## Step 3 — Expanding the Project

After the first result, I asked Lovable to make the app feel more like a middle-sized pet project.

I wanted more than one page or one simple feature.

The app was expanded into sections such as:

* Dashboard
* Events
* Members
* Announcements
* Calendar

## Step 4 — Improving the Dashboard

I refined the dashboard so it had useful overview cards.

The dashboard included things like upcoming events, member count and announcements.

This made the project easier to understand visually.

## Step 5 — Reviewing the Result

I reviewed the generated Lovable result and saved screenshots.

I focused on showing the process, not only the final screen.

The screenshots are used as evidence that I actually used Lovable for this task.

## Step 6 — Relation to Task A

Task A gave me the first GUI idea using Google Stitch.

Task B used Lovable to continue and expand that idea into a larger pet project.

## Short Reflection

Lovable was useful because it quickly turned the project idea into a more complete prototype.

It helped me think about the app as a bigger project with multiple sections.

The result is not my final distributed app. It is my Task B evidence for a vibe-coded pet project.
